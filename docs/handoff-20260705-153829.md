# Handoff: 训练数据重建与 squadValue 单位转换

## Current state

本会话完成了三项核心数据层改造：

### 1. squadValue 单位转换（百万欧元 → 亿欧元）
- `backend/data/teams.json`：批量将 squadValue 除以 100（如 1100 → 11.0）
- `backend/features.py`：归一化系数从 `1100.0` 改为 `11.0`，默认值从 `50` 改为 `0.5`
- `backend/model.py`：缺失球队默认值从 `100` 改为 `1.0`

### 2. historical.csv 完全替换为真实数据（117 场 → 294 场）
- **数据源**：GitHub `openfootball/worldcup.json` + `openfootball/euro.json`（footballcsv.github.io 上游）
- **覆盖范围**：2014/2018/2022 世界杯 + 2021/2024 欧洲杯完整赛事
- **关键修正**：所有点球大战胜负的 `result` 字段已正确处理（有点球 `p` 字段时以点球胜方为准）
- **处理脚本**：临时 `convert_data.py`（含完整英中球队名映射 + 点球/加时判定逻辑），已清理

### 3. historical_data.xlsx 数据整合（294 场 → 395 场）
- 合并了用户提供的 76 场新比赛数据：
  - 2020-2022 友谊赛/欧国联：5 场（葡萄牙 vs 西班牙多场历史对战）
  - 2025 欧国联：1 场（葡萄牙 7-5 西班牙）
  - 2026 世界杯小组赛 + 16 强淘汰赛：70 场（14 支球队的完整参赛数据）
- 日期格式统一：`2025-06-09` → `2025/06/09`
- 去重逻辑：基于 (date, home_team, away_team) 键值合并

### 4. 文件编码修复
- 修复了 `historical.csv` 在用户手动编辑后变为 GBK 编码的问题
- 合并脚本支持 UTF-8/GBK 自动探测，输出始终为 UTF-8

### 5. 模型效果
| 指标 | 改造前 | 改造后 |
|------|--------|--------|
| 训练数据量 | 117 场 | 395 场 |
| 交叉验证准确率 | 80.18%（过拟合） | 78.99%（更可信） |
| squadValue 单位 | 百万欧元 | 亿欧元 |
