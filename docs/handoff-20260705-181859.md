# Handoff: 淘汰赛晋级图可视化重构 + 预测数据刷新

## Current state

本会话聚焦于前端可视化呈现，将淘汰赛晋级图从简单列式布局升级为 SVG 连线树形对阵图，并重新运行后端生成最新预测数据。

### 1. KnockoutProgress 组件重构（`frontend/src/components/KnockoutProgress.tsx`）
- 全新树形对阵图布局：水平 flex 列 + SVG 连线连接器
- `MatchCard` 子组件：已完成比赛显示比分（胜者绿色高亮 + 点球标注），未完成的显示胜率百分比
- `ConnectorCol` 子组件：基于 `ResizeObserver` + DOM 坐标计算，用 SVG `<path>` 绘制轮次间的对阵连线
- `FEEDER_MAP` 硬编码 16 强→8 强→4 强→决赛的映射关系，驱动连线计算
- 季军赛/冠军赛不绘制连线（其对阵方由决赛负者推导）

### 2. Bracket CSS 样式（`frontend/src/styles/global.css`）
- 新增 `.bracket-scroll`、`.bracket`、`.bracket-round`、`.round-header`、`.round-matches` 布局类
- 新增 `.match-card`（含 `.done` 状态）、`.match-team`（含 `.winner`/`.tbd` 状态）、`.bracket-connectors` 样式
- 颜色系统复用 CSS 变量（`--accent-green`、`--accent-amber`、`--border` 等）

### 3. App.tsx 章节图标（`frontend/src/App.tsx`）
- 引入 `lucide-react` 图标：Trophy（夺冠概率）、GitBranch（晋级图）、Target（射手）、BarChart3（准确率）
- 每个 `<h2>` 标题前添加对应彩色图标

### 4. 预测数据刷新
