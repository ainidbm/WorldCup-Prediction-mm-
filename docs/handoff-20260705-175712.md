# Handoff: 淘汰赛实况改造 + 模型准确率扩展

## Current state

本会话完成了 4 项核心改造，将预测系统从"赛前预测"升级为"实时跟踪"：

### 1. 蒙特卡洛模拟接入真实赛果（`backend/monte_carlo.py`）
- `_simulate_one_tournament` 新增 `actual_results` 参数，优先使用 `knockout_bracket.json` 中已有的 `winner` 字段
- 新增 `compute_remaining_matches()` 函数，基于条件概率计算每队剩余场次
- 当前 18 场真实赛果已接入（16 场 R16 + 2 场 QF：摩洛哥、法国晋级）

### 2. 夺冠排行过滤已淘汰队伍（`backend/output.py`）
- `championProb` 过滤 `prob == 0` 的已淘汰队伍（32 队 → 14 队）
- `stageProb` 同步排除已淘汰队伍
- 新增 `_build_knockout_bracket()` 函数，输出完整淘汰赛晋级图数据（6 轮：16强→8强→4强→决赛→季军赛→冠军赛）

### 3. 射手预测修复（`backend/top_scorer.py`）
- **剩余场次**：从累积概率改为条件概率（基于实际赛果确定当前阶段）
- **射门转化率**：0.12 → 0.15（淘汰赛精英射手偏高）
- **出场概率**：0.85 → 0.95（淘汰赛核心球员几乎必上场）
- `predictedGoals` 改为整数输出（7.9 → 8）
- 已淘汰队伍不参与射手排行

### 4. 模型准确率扩展至 90 场（`backend/output.py`）
- `generate_accuracy_json` 新增 `bracket` 参数，淘汰赛已完成比赛纳入回测
- 新增字段：`knockoutAccuracy`、`overallAccuracy`、`groupMatches`、`knockoutMatches`
- 当前结果：综合 74.4%（67/90），小组赛 70.8%（72 场），淘汰赛 88.9%（18 场）

### 5. 前端改造
- 新建 `KnockoutProgress.tsx`：淘汰赛晋级图（水平滚动列式布局，已完成显示比分+胜者高亮，未完成显示概率）
- 删除 `BracketView.tsx` + `MatchCard.tsx`（功能重复）
- `AccuracyPanel.tsx`：新增综合准确率、淘汰赛准确率、交叉验证准确率 4 格展示
- `ChampionBar` 懒加载 + Vite 手动分包（初始 547KB → ~193KB）

### 6. looking_ahead.xlsx
- 用户提供了 8 强对阵分析数据（身价/伤病/战术/教练等定性信息）
- 目前未接入模型，作为后续优化方向
