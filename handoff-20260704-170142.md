# Handoff: 世界杯冠军预测 Agent — 设计决策定稿

## Current state

完成了对世界杯预测 Agent 方案的全面设计拷问（grill-me），所有决策分支已解决，无遗留"待定"项。

### 已确定的关键决策

- **架构**：本地 Python 后端预计算 + GitHub Actions 自动部署 + GitHub Pages 静态前端（Vite + React + Recharts）
- **预测引擎**：随机森林（单场胜/平/负概率）+ 蒙特卡洛模拟（10000 次赛程）；放弃神经网络（数据量不足）
- **方法论**：4 维固定权重（近期状态 40% / 硬实力 30% / 交锋 15% / 情境 15%）仅作可解释性层，不参与概率计算；胜率硬上限 85%
- **数据源**：Football-Data.co.uk（历史）+ Transfermarkt/FBref（伤病与交锋）+ 博彩赔率（仅模型输入，不向用户展示）；砍掉专家意见；反爬失败降级为手动更新
- **赛程状态**：小组赛已结束，16 强对阵表硬编码到代码中；小组赛全部结果已收集作为训练/校准数据
- **输出契约**：`predictions.json`（全中文字段）含夺冠概率排行、各阶段晋级概率、16 场对阵预测、最佳射手 Top 10；`accuracy.json` 用于模型校准
- **射手预测**：简单加权公式（已进球 + 剩余场次 × 射门转化率 × 出场概率），输出 Top 10
- **部署流程**：手动编辑 `knowledge_base.json` / `results.json` → git push → GitHub Actions 自动跑 Python → 生成 JSON → 部署 Pages；支持 push 触发 + 每日定时兜底 + workflow_dispatch 手动触发
- **已砍功能**：神经网络、专家意见、黑马排行、多语言、实时交互
