### 1.2 数据采集能力

#### 1.2.1 数据源与数据维度

系统采集并整合了以下多维度世界杯数据：

| 数据文件 | 内容 | 数据量 | 关键字段 |
|---------|------|--------|---------|
| `teams.json` | 14 支存活球队档案 | 14 队 | FIFA 排名、球员身价、核心球员、历史战绩、平均年龄、东道主标记 |
| `group_results.json` | 小组赛完整赛果 | 72 场 | 对阵双方、比分、进球者名单 |
| `knockout_bracket.json` | 淘汰赛对阵图 | 16→1 场 | 16 强对阵、晋级结构、已完成比赛赛果（含点球） |
| `historical.csv` | 历史世界杯赛事 | 395 场 | 日期、主客队、比分、胜负结果（2014-2022 三届世界杯） |

#### 1.2.2 数据覆盖维度

每支球队的数据档案包含 **7 个维度**：

- **基础属性**：球队分层（top/first/second/host）、FIFA 排名
- **阵容价值**：球队总身价（亿欧元）、核心球员名单（含 47 名球员射门数据）
- **历史底蕴**：世界杯夺冠次数、历史最佳成绩、上届世界杯表现
- **近期状态**：小组赛积分、进球数、失球数、净胜球、胜负记录
- **情境因素**：东道主标记（美国/加拿大/墨西哥）、平均年龄
- **交锋记录**：从 395 场历史数据中提取的双队交锋胜/平/负比率
- **射手统计**：小组赛 + 淘汰赛已进球球员的逐场进球记录

### 1.3 分析决策能力 — 预测逻辑与决策链路

#### 1.3.1 特征工程：4 维可解释特征体系

系统设计了 **4 维特征向量**，每维均归一化至 0-1 区间，确保可解释性：

| 特征维度 | 权重 | 计算方法 | 可解释性 |
|---------|------|---------|---------|
| **近期状态 (form)** | 40% | 小组赛积分(50%) + 净胜球(30%) + 进球数(20%) | "近期状态更佳" |
| **硬实力 (strength)** | 30% | FIFA排名(40%) + 球员身价(35%) + 历史夺冠(25%) | "整体实力占优" |
| **交锋记录 (h2h)** | 15% | 历史对阵胜/平/负概率 | "交锋心理优势" |
| **情境因素 (context)** | 15% | 东道主(40%) + 大赛经验(35%) + 年龄结构(25%) | "东道主主场优势" |

每场比赛生成 10 维特征向量输入模型（4 维差值 + 6 维绝对值），同时生成中文关键因素列表用于前端展示。

#### 1.3.2 随机森林模型

- **算法**：`RandomForestClassifier`（200 棵树，最大深度 10）
- **训练数据**：395 场历史世界杯赛事（2014/2018/2022 三届）
- **交叉验证**：5 折交叉验证，准确率 **78.99%**
- **胜率上限**：硬编码 85% 上限，超出部分按比例分配给平局和另一方，避免极端预测
- **输出**：对任意两队返回 (胜A概率, 平局概率, 胜B概率) 三元组

```
模型参数:
  n_estimators = 200
  max_depth = 10
  min_samples_split = 20
  min_samples_leaf = 10
  random_state = 42
```

#### 1.3.3 蒙特卡洛模拟（10,000 次）

这是系统的**核心推理引擎**，通过大规模随机模拟推演完整赛程：

**算法流程**：

1. **预计算**：对所有存活球队的两两组合预计算胜率（14 队 = 91 组对阵），避免循环内重复调用模型
2. **单次模拟**：从 16 强 → 8 强 → 4 强 → 决赛 → 冠军赛，逐场模拟
   - 已完成比赛：直接使用真实赛果
   - 未完成比赛：按胜率概率随机抽样决定胜者
   - 平局处理：模拟加时/点球（各 50% 概率）
3. **统计聚合**：10,000 次模拟后，统计每支球队在 5 个阶段的晋级频率作为概率

**赛果融合机制**：
- 系统支持**渐进式赛果注入** — 随着真实比赛进行，已完赛结果会锁定到模拟中，剩余模拟仅推演未完成比赛
- 当前状态：16 场 R16 + 2 场 QF 已完成，真实赛果已注入

#### 1.3.4 比分预测算法（Poisson MAP + 平局修正）

在胜率概率基础上，系统进一步预测具体比分：

**算法原理**：

```
步骤 1：计算预期进球 (xG)
  xg_a = (A 场均进球 × 0.55 + B 场均失球 × 0.45) × 0.85
  xg_b = (B 场均进球 × 0.55 + A 场均失球 × 0.45) × 0.85

步骤 2：胜率概率混合
  xg_a = xg_a × 0.30 + ratio_a × 3.5 × 0.70
  xg_b = xg_b × 0.30 + ratio_b × 3.5 × 0.70

步骤 3：总进球下限保证
  if xg_total < 2.5: 按比例放大至 2.5

步骤 4：强弱悬殊非对称放大
  if ratio > 0.75: 强队 ×1.15, 弱队 ×0.80

步骤 5：Poisson MAP 搜索
  遍历 0-6 球所有组合，选概率最大的 (sa, sb)

步骤 6：平局修正
  if 平局 and 胜率差 > 15%: 强者 +1 球
```

**设计理由**：淘汰赛场均 2.5 球，纯 Poisson MAP 在低 lambda 时过度保守，通过提高基线 + 概率主导混合 + 平局修正确保比分多样性。

#### 1.3.5 射手预测

采用加权公式预测最佳射手 Top 10：

```
预测总进球 = 已进球 + 剩余场次 × 场均射门 × 转化率(0.15) × 出场概率(0.95)
```

- **剩余场次**：由蒙特卡洛模拟的条件晋级概率计算（非简单估算）
- **射门数据**：47 名核心球员的逐人场均射门数
- **数据来源**：小组赛 + 淘汰赛已进球记录自动提取，过滤乌龙球和点球大战进球

## 三、创新与创意（20 分）

### 3.1 预测逻辑创新

#### 创新点 1：渐进式赛果融合

不同于一次性预测全部赛程，系统支持**渐进式赛果注入**：

- 已完成比赛的真实赛果锁定到蒙特卡洛模拟中
- 10,000 次模拟仅推演未完成比赛
- 随着赛事推进，预测精度持续提升
- 当前已注入 18 场真实赛果（16 场 R16 + 2 场 QF）

这一设计使系统从"赛前预测"进化为"赛中实时预测"，每次运行都反映最新赛况。

#### 创新点 2：Poisson MAP + 平局修正的比分预测

传统方法要么不预测比分，要么用纯 Poisson MAP（容易过度保守）。系统创新性地：

1. 将**小组赛统计数据**与**蒙特卡洛胜率概率**混合（30:70 权重）
2. 引入**总进球下限**保证比赛观赏性预期
3. 对**强弱悬殊**进行非对称放大（强队 ×1.15，弱队 ×0.80）
4. **平局修正**机制：当 Poisson MAP 给出平局但胜率差异 >15% 时，自动给强者加 1 球

这一组合产生了 2-1、1-2、0-2、2-0、3-0 等多样化比分，而非全是 1-0/0-1。

#### 创新点 3：条件概率剩余场次计算

射手预测中的"剩余场次"不是简单估算，而是基于蒙特卡洛模拟的条件概率：

```
剩余场次 = 1(保证当前轮次) + P(晋级下一轮|当前轮) + P(晋级再下一轮|当前轮)
```

已淘汰球队剩余场次为 0，自动从射手榜中移除。

### 3.2 Agent 设计创新

#### 创新点 4：全自动 CI/CD 管线

系统实现了**零人工干预**的自动更新管线：

```
push to main → GitHub Actions
  → Python 3.12 + pip install
  → python backend/main.py (运行预测引擎)
  → Node.js 20 + npm ci
  → npm run build (Vite 构建)
  → cp predictions.json + accuracy.json → deploy/
  → GitHub Pages 部署
```

- 每日 UTC 06:00 定时兜底运行
- 支持手动触发 (workflow_dispatch)
- 更新赛果数据后只需 push，全链路自动完成

#### 创新点 5：可解释性双层设计

系统在**模型层**和**展示层**均提供可解释性：

- **模型层**：4 维特征体系（状态/实力/交锋/情境），每维有明确权重和计算逻辑
- **展示层**：中文关键因素列表直接展示在比赛预测旁，用户无需理解模型即可理解推理

### 3.3 额外功能亮点

#### 亮点 1：淘汰赛晋级图 SVG 动态连线

使用 React Ref + ResizeObserver + SVG path 动态计算连线坐标，实现：

- 窗口缩放时连线自动重新计算

淘汰赛已完成比赛中，点球大战结果以 `(3)` 格式附加在常规比分旁，完整还原真实赛果。

#### 亮点 3：多维度准确率回测

不仅展示模型交叉验证准确率，还提供：

- 淘汰赛独立验证集准确率（18 场）
- 综合准确率（90 场）
- 回测范围和方法论说明文字
